## 🔑 КАК ПОЛУЧИТЬ КЛЮЧИ

### BOT_TOKEN
1. Открыть [@BotFather](https://t.me/BotFather)
2. Отправить `/newbot`
3. Следовать инструкциям
4. Скопировать токен

### GROQ_API_KEY (Рекомендуется)
1. Зарегистрироваться на [console.groq.com](https://console.groq.com)
2. Перейти в API Keys
3. Создать новый ключ
4. Скопировать (начинается с `gsk_...`)

### GEMINI_API_KEY (Альтернатива)
1. Зарегистрироваться на [aistudio.google.com](https://aistudio.google.com/app/apikey)
2. Создать API ключ
3. Скопировать

### TG_API_ID и TG_API_HASH (Для UserBot)
1. Открыть [my.telegram.org](https://my.telegram.org)
2. Войти со своим номером телефона
3. Нажать **API development tools**
4. Создать приложение (любое название)
5. Скопировать **App api_id** и **App api_hash**

## 🐛 РЕШЕНИЕ ПРОБЛЕМ

### ❌ AI не работает (крестик в логах)

**Проблема**: `[AI] Groq: ❌ | Gemini: ❌`

**Решение**:
1. Проверьте `.env` файл - ключи должны быть БЕЗ кавычек:
   ```env
   GROQ_API_KEY=gsk_ваш_ключ  # ✅ ПРАВИЛЬНО
   GROQ_API_KEY="gsk_ваш_ключ"  # ❌ НЕПРАВИЛЬНО
   ```

2. Проверьте валидность ключей:
   ```bash
   # Для Groq:
   curl https://api.groq.com/openai/v1/models \
     -H "Authorization: Bearer ВАШ_GROQ_KEY"
   
   # Должен вернуть список моделей
   ```

3. Убедитесь что `.env` файл находится в корне проекта (рядом с `index.js`)

### ⏱ Таймер-медиа не скачиваются

**Проблема**: Файлы с таймером не сохраняются

**Решение**:
1. Проверьте права на папку `media/`:
   ```bash
   chmod -R 755 media/
   ```

2. Проверьте логи - должны быть сообщения:
   ```
   [MEDIA] ✅ Скачано: media/123/timer_photo_...
   ```

3. Убедитесь что в настройках включено:
   ```
   ⚙️ Настройки → Таймер-медиа: ✅
   ```

### 🤖 UserBot не подключается

**Проблема**: `[UB] UserBot: ❌ Не настроен`

**Решение**:
1. Проверьте `.env`:
   ```env
   TG_API_ID=12345678  # ТОЛЬКО ЦИФРЫ, без кавычек
   TG_API_HASH=abcd1234  # БЕЗ КАВЫЧЕК
   ```

2. Убедитесь что credentials с [my.telegram.org](https://my.telegram.org)

3. При подключении вводите номер в формате `+79991234567`

### 📊 Неправильное количество пользователей

**Решение**: В этой версии статистика берется напрямую из БД:
```sql
SELECT COUNT(*) FROM users;
```

Должно показывать точное количество (11,736 согласно БД).
